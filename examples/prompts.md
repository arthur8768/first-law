# Example Prompts

这些示例用于触发 `$first-law`。可以直接替换主题、产品或决策背景。

## 概念澄清

```text
使用 $first-law，从第一性原理解释“信任”。请区分信任、可靠、熟悉和喜欢，并给出一个可证伪条件。
```

```text
使用 $first-law，分析“执行力”的本质。不要用同义词定义，也不要直接套用目标管理框架。
```

## 产品与 MVP

```text
使用 $first-law，分析下面这个产品为什么需要存在、解决的不可约问题、最小必要功能和最便宜的验证实验：
[粘贴产品想法]
```

```text
使用 $first-law，检查这个 MVP 是否真的最小。对每项功能做删除测试，并标注必要、使能、增强或结果变量：
[粘贴功能列表]
```

## 商业模式

```text
使用 $first-law，拆解这个商业模式成立所需的最小系统、关键约束、因果机制和失败条件：
[粘贴商业模式]
```

```text
使用 $first-law，分析“免费增值模式”的本质。区分获客机制、付费机制和留存结果，并给出可观察预测。
```

## 组织与管理

```text
使用 $first-law，分析一个五人团队为什么需要项目管理。请从协调成本、不确定性、责任和反馈出发推导，不预设敏捷或瀑布。
```

```text
使用 $first-law，判断这项公司制度是必要机制、使能条件还是历史惯例：
[粘贴制度]
```

## 决策与个人成长

```text
使用 $first-law，分析我是否应该辞职创业。把事实、假设和推论分开，并给出能最大幅度降低不确定性的小实验：
[补充背景]
```

```text
使用 $first-law，拆解“长期主义”在个人职业决策中的成立条件、适用边界和可能造成的二阶成本。
```

## 研究模式

```text
使用 $first-law 和 $deep-research，研究远程办公对知识工作团队绩效的影响。先列来源支持的发现，再做第一性原理综合，并说明哪些证据会推翻结论。
```
