# first-law | 第一性原理分析 Skill

一个帮助 AI 从可辩护的前提、约束和因果机制出发，拆解复杂概念、产品、组织系统与商业命题的 Codex Skill。

它不是把问题换一种说法，也不是套用熟悉的商业框架。它要求分析者明确对象与边界，找到对象存在所解决的不可约问题，推导最小充分系统，并通过反例、失败条件和可证伪预测检查结论。

## 核心能力

- 区分事实、用户给定、假设、推论与分析者综合
- 从不可约问题和反事实出发解释对象为什么存在
- 识别稀缺性、不确定性、激励、成本、时间与反馈等约束
- 构建从输入、机制、输出到结果的因果链
- 使用删除测试和充分性测试寻找最小系统
- 区分相邻概念、输入、使能条件、输出与结果
- 检查反例、替代解释、权衡与失败边界
- 给出可观察指标、预测和证伪条件
- 把推导结果转成产品、商业或个人决策

## 三种分析模式

| 模式 | 适用场景 |
| --- | --- |
| Quick | 低风险的概念澄清，尽量简洁 |
| Standard | 产品、商业、组织、教学等常规分析 |
| Research | 有争议、时效性强、依赖经验事实或高风险的命题 |

Research 模式会调用 `$deep-research`，并将有来源支持的发现与第一性原理综合分开呈现。

## 分析流程

```text
定义对象与范围
  -> 暂停沿用现成标签
  -> 找到不可约问题
  -> 暴露约束
  -> 推导因果机制
  -> 构建最小充分系统
  -> 划定边界
  -> 压力测试
  -> 操作化与证伪
  -> 转化为行动
```

## 安装

使用 Skills CLI：

```bash
npx skills add https://github.com/arthur8768/first-law --skill first-law
```

或直接克隆到 Codex Skills 目录：

```bash
git clone https://github.com/arthur8768/first-law.git ~/.codex/skills/first-law
```

如果目标目录已存在，请先备份或选择其他安装目录，不要直接覆盖个人修改。

## 使用示例

```text
使用 $first-law，从第一性原理分析“一人公司”的本质、最小必要系统和失败条件。
```

```text
使用 $first-law，判断这个 AI 产品功能是真需求还是伪需求，并给出可证伪的 MVP 实验。
```

```text
使用 $first-law，拆解“品牌”的不可约功能，并区分品牌、营销、口碑和知名度。
```

```text
使用 $first-law，分析团队为什么需要项目管理。不要套用敏捷框架，先推导必要条件。
```

```text
使用 $first-law 和 $deep-research，研究订阅制商业模式成立的前提、边界与当前证据。
```

更多示例见 [`examples/prompts.md`](examples/prompts.md)。

## 输出内容

一次完整分析通常会回答：

1. 分析对象到底是什么；
2. 它为什么需要存在；
3. 它通过什么机制发挥作用；
4. 哪些组成部分必要，哪些条件足以让系统成立；
5. 它不是什么，与相邻概念如何区分；
6. 它在什么条件下失败；
7. 如何观察、验证或推翻这套解释；
8. 对实际行动有什么影响。

## 设计原则

- 不把类比当作因果证据
- 不把当前惯例称为自然法则
- 不用组件清单代替必要性与充分性证明
- 不把好结果反推为过程一定正确
- 不隐藏结论成立所依赖的假设
- 不输出无法说明什么证据会改变它的模型
